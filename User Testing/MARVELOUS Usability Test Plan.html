Overall Objectives

The overall goal of this test is to check the accessibility and ease of use of the application for both users who are familiar with the MARVEL universe and for users who are not. In particular, we want to focus on the following aspects:

  • Assess the overall effectiveness of the application for different types of users performing basic, common tasks;
  • Identify obstacles to completing a certain task or using a certain component;
  • Evaluate consistency and standards;
  • Check if users identify the main user flows of the application and follow them;
Scenario

You come home after work one day. You want to relax and learn something new. You open the MARVALOUS application and you want to explore it.

Tasks
  • Task 1: Open the main screen of the application
    Estimated time: 0,1 min
  • Task 2: Navigate through the main screen, to get familiar with it
    Estimated time: 1 min
  • Task 3: Navigate to the character "BLACK PANTHER". Read information about him.
    Estimated time: 3 min
  • Task 4: Get back to the mainscreen and go to the year 1945
    Estimated time: 1 min
Post-test Questions

These questions could be presented to the user as a questionnaire or verbally asked.

  • Question 1: How was the overall user experience?
  • Question 2: What did you find most enjoyable?
  • Question 3: What tasks did you have trouble with? What would you improve?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".

Test Results for: Product/Application Component

Success Criteria

A successful design has been achieved when:

  • Under 5% of the users are met with a bug in the application.
  • 70% of the users are able to find the information they need in under 50 seconds.
  • 80% of the users are able to navigate smoothly through the application.
